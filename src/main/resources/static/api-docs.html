Overview

The Wordz API provides endpoints for generating Scrabble puzzles, managing tile bags, and retrieving game information. The API is built with Spring Boot and returns JSON responses.

Note: All endpoints return JSON responses. The server runs on port 8084 by default.

Base URL

http://localhost:8084

Health Check

GET /health
Check if the application is running and healthy.

Request

GET http://localhost:8084/health

Response

"Application is running!"
200 OK - Application is running

Puzzle Endpoints

Endpoints for generating and retrieving Scrabble puzzles with valid word placements.

POST /api/puzzle/generate
Generate a new Scrabble puzzle with 93 tiles placed as valid words on the board, leaving 7 tiles as the puzzle challenge.

Request

POST http://localhost:8084/api/puzzle/generate

Response

{
  "board": {
    "grid": [
      ["A", "B", "C", ...],
      ["D", "E", "F", ...],
      ...
    ],
    "placedTiles": [
      {"row": 7, "col": 5, "letter": "H", "points": 4},
      {"row": 7, "col": 6, "letter": "E", "points": 1},
      ...
    ]
  },
  "remainingTiles": [
    {"letter": "A", "points": 1},
    {"letter": "E", "points": 1},
    ...
  ],
  "placedTileCount": 93,
  "remainingTileCount": 7
}
200 OK - Puzzle generated successfully
GET /api/puzzle/current
Retrieve the currently generated puzzle without creating a new one.

Request

GET http://localhost:8084/api/puzzle/current

Response

{
  "board": {
    "grid": [...],
    "placedTiles": [...]
  },
  "remainingTiles": [...],
  "placedTileCount": 93,
  "remainingTileCount": 7
}
200 OK - Current puzzle retrieved 404 Not Found - No puzzle generated yet

Scrabble Game Endpoints

Endpoints for managing the Scrabble tile bag and retrieving game information.

GET /api/scrabble/bag/info
Get information about the current state of the Scrabble tile bag.

Request

GET http://localhost:8084/api/scrabble/bag/info

Response

{
  "remainingTiles": 100,
  "isEmpty": false
}
200 OK - Bag info retrieved
POST /api/scrabble/bag/draw
Draw multiple tiles from the bag.

Query Parameters

Parameter Type Required Default Description
count integer No 7 Number of tiles to draw

Request

POST http://localhost:8084/api/scrabble/bag/draw?count=5

Response

{
  "drawnTiles": ["A", "E", "R", "S", "T"],
  "tileDetails": [
    {"letter": "A", "points": 1},
    {"letter": "E", "points": 1},
    {"letter": "R", "points": 1},
    {"letter": "S", "points": 1},
    {"letter": "T", "points": 1}
  ],
  "remainingTiles": 95
}
200 OK - Tiles drawn successfully
POST /api/scrabble/bag/draw-single
Draw a single tile from the bag.

Request

POST http://localhost:8084/api/scrabble/bag/draw-single

Response

{
  "letter": "Q",
  "points": 10,
  "remainingTiles": 99
}
200 OK - Tile drawn successfully
POST /api/scrabble/bag/reset
Reset the bag to its initial state with all 100 tiles.

Request

POST http://localhost:8084/api/scrabble/bag/reset

Response

{
  "message": "Bag reset successfully",
  "remainingTiles": 100
}
200 OK - Bag reset successfully
GET /api/scrabble/bag/shuffle
Shuffle the remaining tiles in the bag.

Request

GET http://localhost:8084/api/scrabble/bag/shuffle

Response

{
  "message": "Bag shuffled successfully",
  "remainingTiles": 93
}
200 OK - Bag shuffled successfully
GET /api/scrabble/points/{letter}
Get the point value and count for a specific letter in the standard Scrabble distribution.

Path Parameters

Parameter Type Required Description

Request

GET http://localhost:8084/api/scrabble/points/Q

Response

{
  "letter": "Q",
  "points": 10,
  "count": 1
}

Response Formats

Board State

The board state is returned as a JSON object containing:

  • grid: 15x15 array representing the board layout
  • placedTiles: Array of tile objects with position and letter information

Tile Objects

Tiles are represented as objects with:

  • letter: The letter on the tile (A-Z)
  • points: Point value of the tile
  • row: Board row position (0-14)
  • col: Board column position (0-14)

Puzzle Response

Puzzle responses include:

  • board: Current board state
  • remainingTiles: Array of tiles not yet placed
  • placedTileCount: Number of tiles on the board
  • remainingTileCount: Number of tiles remaining

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of requests.

200 OK - Request successful 400 Bad Request - Invalid request parameters 404 Not Found - Resource not found 500 Internal Server Error - Server error
Note: Most endpoints return detailed error messages in the response body when errors occur.

Getting Started

To get started with the Wordz API:

  1. Ensure the application is running on http://localhost:8084
  2. Test the health endpoint: GET /health
  3. Generate your first puzzle: POST /api/puzzle/generate
  4. Explore the Scrabble bag management endpoints
